◆ chunk_file_store()

int chunk_file_store ( chunk_t  chunk,
char const *  programmPath,
chunk_file_compile_flag  flags 
)

Stores the chunks that were generated by compiling a cellox program as a file.

A cellox bytecode file has the following structure

  1. -—file metadata---—
  2. -—top level chunk-—

The metadata of a file starts with the compileflags stored in a single byte. It is followed by a byte indicating the major and another byte indicating the minor version of cellox that was used to create the chunk. Each chunk is stored in three different segments:

  1. -—constants--—
  2. -—lineInfo---—
  3. -—inner chunks–
  4. -—bytecode---—

The inner chunks segment contains all the functions that were declared in the parent chunk.

Parameters
chunkThe chunk that is stored
programmPathThe program path of the cellox program
flagsCompile flags used to compile the sourcecode to a cellox chunk file
Returns
0 -> OK, -1 -> Error
